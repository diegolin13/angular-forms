Características del formulario

Validaciones

El siguiente formulario cuenta con 3 tipos de validaciones:

  1. Validaciones síncronas:
    Todos los campos en el formulario son requeridos. Adicionalmente, para el nombre y apellidos se necesitan mínimo 3 letras y para validar el correo se utiliza una expresión regular.

  2. Validaciones asíncronas:
    El campo de correo cuenta también con una validación asíncrona que se termina de ejecutar después de 2 segundos al detectar cambios en su valor. Con esto se simula que al ingresar el correo, se evalué si ya ha sido registrado con anterioridad. (Aunque para este ejercicio solo existe un correo que dispara el error "test@test.com" 🙊).

  3. Validaciones personalizadas:
    Se creó una validación personalizada para el campo de Hobby. En esta validación, se prohíbe que el usuario ingrese la cadena de texto "patear perritos". En caso de detectarse, el formulario queda invalidado.

FormGroupName

Este formulario cuenta con dos campos que estan agrupados en un solo objeto, los cuales son: Calle y Ciudad mismos que se ecuentran dentro del objeto domicilio. De esta manera al obtener el valor del formulario encontraremos una propiedad de nombre domicilio la cual contendrá una calle y una ciudad

Form Array

Se utilizó un FormArray para que el usuario pueda agregar y quitar la cantidad de hobbies que desee. Cada control nuevo generado se maneja de forma independiente.

Mensajes de error

Cuando el valor de algún campo choque con alguna validación, se muestra un mensaje de retroalimentación para que el usuario pueda corregirlo.
